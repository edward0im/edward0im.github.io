1 Introduction

리눅스에서 프로그램을 빌드하다가 한 번쯤은 버전 문제로 제대로 실행하지 못한 경험이 있을 것이다. Docker는 이러한 빌드 문제를 해결해줄 수 있는 좋은 솔루션이다. 필자 또한 빌드 문제로 많은 시간을 낭비하다가 결국 Docker를 사용하기 시작했다. 해당 포스트는 필자가 docker를 사용하면서 필요한 정보들을 정리해놓은 포스트이다.

2 Installation

Ubuntu 16.04 LTS 환경에서 docker는 다음 명령어를 통해 간단하게 설치할 수 있다. 

$ sudo apt-get install docker*

3 Docker commands

가장 자주 사용하는 docker 명령어들을 정리하였다. 

# 특정 컨테이너 삭제
$ docker rm {...}

# 특정 이미지 삭제
$ docker rmi {...}

# 컨테이너 목록 검색
$ docker ps -a

# 이미지 목록 검색
$ docker images

# 특정 컨테이너에 접속
$ docker attach {...}

# docker run은 터미널에서 바로 실행하기보다 보통 shell script (.sh) 파일로 저장하여 한 번에 실행한다.
$ docker run -it --name ros -v ~/docker:/root/data edward:starlaw
$ docker run -i -t --name hello ubuntu:16.04 /bin/bash
$ docker run -i --net host --name hello -t ubuntu:16.04 /bin/bash
# i : interactive
# t : pseudo-tty
# --net :
# --name : 이름

# ros_kinetic이라는 컨테이너를 edward:starlaw 이미지로 저장한다. 작성자는 edward0im, 메세지는 {...}
$ docker commit -a "edward0im" -m "catkin_ws installed" ros_kinetic edward:starlaw

# 원격 dockerhub에서 특정 이미지를 받아옴
$ docker pull ubuntu:16.04

4 w/o sudo command

우분투에서 docker를 처음 설치하고 실행하면 매번 실행할 때마다 sudo 명령어를 입력해줘야 한다. 이를 sudo 명령어없이 편하게 사용하려면 현재 계정을 docker 그룹에 포함시키면 된다. 명령어는 아래와 같다. 

$ sudo usermod -aG docker ${USER}
$ sudo service docker restart
# 현재 계정을 로그아웃하고 재접속하면 적용된다

5 docker service

# 서비스 동작 상태 확인
$ sudo service docker status

6 nvidia-docker

nvidia-docker 는 호스트 컴퓨터에 NVIDIA 드라이버가 있을 경우 컨테이너에서 GPU를 사용할 수 있도록 해주는 프로그램이다.

6.1 Installation

  • 자세한 내용은 여기에 소개되어 있다. 

    # Add the package repositories
$ curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
$ distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
$ curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | sudo tee /etc/apt/sources.list.d/nvidia-docker.list
$ sudo apt-get update

# Install nvidia-docker2 and reload the Docker daemon configuration
$ sudo apt-get install -y nvidia-docker2
$ sudo pkill -SIGHUP dockerd

# Test nvidia-smi with the latest official CUDA image
$ docker run --runtime=nvidia --rm nvidia/cuda:9.0-base nvidia-smi

    만약 설치 도중 docker-ce 의존성 에러가 나는 경우 다음 코드를 추가적으로 실행해준다. 아래 코드는 여기에서 가져왔다. 

    $ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
$ sudo apt-key fingerprint 0EBFCD88
$ sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable
$ sudo apt-get update
$ sudo apt-get install docker-ce
$ sudo service docker restart

6.2 Usage

nvidia-docker 명령어는 docker run –runtime=nvidia 와 동일하다. 만약 non-cuda 이미지에서 GPU를 사용하고 싶을 경우 아래와 같은 명령어를 사용하면 된다. 출처는 여기이다. 

# -e NVIDIA_VISIBLE_DEVICES=all 을 입력하면 non-CUDA 이미지에서도 gpu를 사용할 수 있는 듯
$ docker run --runtime=nvidia -e NVIDIA_VISIBLE_DEVICES=all -it --name ros_nvidia2 edward:starlaw

7 Docker Hub

docker도 github처럼 사이트를 운영하고 있다. 여기에 가입하고 명령어를 통해 터미널에서 로그인할 수 있다. 

$ docker login
$ docker tag edward:starlaw edward0im/docker
$ docker push edward0im/docker

간단하게 정리한 docker hub push 명령어는 아래와 같다 

# tag를 달고
$ docker tag edward0im:ubuntu_cuda9.0 edward0im/ubuntu_cuda9.0
# dockerhub에 push한다
$ docker push edward0im/ubuntu_cuda9.0

8 Connect to internet in docker container

Docker 컨테이너는 NAT 기술을 사용하여 외부 네트워크로 통신한다. 실제 인터넷 선은 하나 밖에 제공되어 있지 않고 그 선을 통해 데이터가 나가야 하기 때문에 Linux의 iptables 명령을 사용해서 해당 브릿지 인터페이스에 MASQUERADE 기능을 부여하거나 SNAT 값을 줘야 한다. 해당 방법은 이 블로그를 통해 얻은 정보이다. 

  # host 머신에서 아래 명령어를 실행한다. 아래 명령어를 통해 iptables에 NAT 정보를 입력한다
  $ sudo iptables -t nat -A PREROUTING -d 0.0.0.0/0 -m addrtype --dst-type LOCAL -j DOCKER
  $ sudo iptables -t nat -A OUTPUT -m addrtype --dst-type LOCAL -j DOCKER
  $ sudo iptables -t nat -A POSTROUTING -s 172.17.0.0/16 -j MASQUERADE

  # 명령어가 정상적으로 작동했는지 확인한다.
  $ sudo iptables -t nat -L

Chain PREROUTING (policy ACCEPT)
target     prot opt source               destination         
DOCKER     all  --  anywhere             anywhere             ADDRTYPE match dst-type LOCAL
DOCKER     all  --  anywhere             anywhere             ADDRTYPE match dst-type LOCAL

Chain INPUT (policy ACCEPT)
target     prot opt source               destination         

Chain OUTPUT (policy ACCEPT)
target     prot opt source               destination         
DOCKER     all  --  anywhere            !127.0.0.0/8          ADDRTYPE match dst-type LOCAL
DOCKER     all  --  anywhere             anywhere             ADDRTYPE match dst-type LOCAL

Chain POSTROUTING (policy ACCEPT)
target     prot opt source               destination         
MASQUERADE  all  --  172.17.0.0/16        anywhere            
MASQUERADE  all  --  172.17.0.0/16        anywhere            

Chain DOCKER (4 references)
target     prot opt source               destination         
RETURN     all  --  anywhere             anywhere

9 Bash autocompletion in docker

Docker에 접속하면 기본적으로 bash에서 tab 자동 완성(autocompletion)이 작동하지 않는다. 아래 작성한 방법들 모두 적용해본 결과 정상적으로 작동하는 걸 확인했다. 정확히 어느 파트는 필요없는 지에 대한 내용은 아직 모르겠다. ROS 명령어들을 자동완성하고 싶은 경우 source /opt/ros/kinetic/setup.bash 를 .bashrc 파일에 넣어줘야 한다. 

  $ apt-get install bash_completion

  # download bash completion file into /etc/bash_completion.d/ 
  $ curl https://raw.githubusercontent.com/docker/docker-ce/master/components/cli/contrib/completion/bash/docker -o /etc/bash_completion.d/docker.sh

  # .bash.bashrc 파일을 열고 
  $ vim /etc/bash.bashrc

# 아래 코드를 주석 해제한다.
if ! shopt -oq posix; then
 if [ -f /usr/share/bash-completion/bash_completion ]; then
   . /usr/share/bash-completion/bash_completion
 elif [ -f /etc/bash_completion ]; then
   . /etc/bash_completion
 fi
fi

  # 해당 코드를 실행한다.
  $ source /etc/bash.bashrc

  # ~/.bashrc 에도 비슷한 completion 코드가 있는데 주석 해제 후 source를 해준다.

10 Dockerfile

Dockerfile은 docker build 명령어를 사용하여 docker 이미지를 생성할 수 있는 파일이다. 아래에 Dockerfile의 간단한 사용 예시를 작성했다. 우선 example 폴더를 만들고 그 안에 아래와 같은 Dockerfile을 생성한다. 

$ mkdir example && cd example
$ vim Dockerfile

# Dockerfile
  FROM ubuntu:16.04   # 어떤 이미지를 기반으로 할지 설정 {이미지이름}:{태그}
  MAINTAINER Foo Bar foo@bar.com # 관리자 정보

  RUN apt-get update  # RUN으로 쉘스크립트 혹은 명령 실행
  RUN apt-get install -y nginx  # 이미지 생성 중에는 사용자 입력을 받을 수 없으므로
  RUN echo "\ndaemon off;" >> /etc/nginx/nginx.conf # apt-get install -y 옵션 사용
  RUN chown -R www-data:www-data /var/lib/nginx

  VOLUME ["/data", "/etc/nginx/site-enabled","/var/log/nginx"] # 호스트와 공유할 디렉토리 목록

  WORKDIR /etc/nginx  # 다음 CMD에서 설정한 실행 파일이 실행될 디렉토리

  CMD ["nginx"] # 컨테이너가 시작되었을 때 실행할 실행 파일 또는 스크립트

  EXPOSE 80   # 호스트와 연결할 포트 번호
  EXPOSE 443

위와 같이 Dockerfile을 작성하고 해당 폴더에서 다음 명령을 실행하면 빌드된다. 

$ docker build --tag hello:0.1 .
# --tag 옵션으로 이미지 이름과 태그를 설정할 수 있다
# 이미지 이름만 설정하면 태그는 latest로 설정된다.

# 잠시 기다리면 이미지가 생성된다. 이미지 목록을 출력하면
$ docker images

# 앞에서 생성한 이미지를 실행해보기
$ sudo docker run --name hello-nginx -d -p 80:80 -v /root/data:/data hello:0.1
# -d : 컨테이너를 백그라운드로 실행
# -p 80:80 : 호스트의 80번 포트와 컨테이너의 80번 포트를 연결하고 외부에 노출
# -v /root/data:/data : 호스트의 /root/data 디렉토리를 컨테이너의 /data 디렉토리에 연결

# 웹 브라우저를 실행하고 http://{IP}:80 으로 접속하면 nginx 페이지가 표시된다. 이렇게 나오면 정상
