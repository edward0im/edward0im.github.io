1 Introduction

본 포스트에서는 리눅스에서 emacs 에디터를 통해 C++/Python 개발환경을 구축하는 방법에 대하여 설명한다. emacs는 40년이 넘는 오랜 역사를 가지고 있는 에디터이므로 매우 다양한 코드 네비게이션 방법들이 존재하지만 본 포스트에서는 lsp-mode 를 사용하여 코드 네비게이션하는 방법에 대해 설명한다.

emacs는 매우 복잡하고 많은 키바인딩으로 유명하다. 이러한 복잡한 키바인딩을 단순화시켜서 편하게 사용하기 위해 필자는 evil 모드를 설치하여 vim 키바인딩을 사용하였으며 대부분의 키바인딩을 Space 키 기반으로 변경하여 손에 부담이 덜 가도록 설정하였다. spacemacs 같이 이미 많은 사람들이 사용하는 Space 키 기반의 emacs가 존재하지만 use-package 기반의 환경설정을 사용할 수 없어서 필자만의 커스텀 Space 키바인딩을 사용하였다.

emacs는 C++ 에디터 외에도 org-mode, latex-mode 등과 같은 강력한 패키지들을 사용하여 일정 관리 및 논문 작성, 블로그 포스팅 등 여러 기능들을 사용할 수 있으나 본 포스트에서는 C++ 개발환경설정에 초점을 맞춘다. 그리고 emacs는 GUI, CLI 모두를 지원하지만 본 포스트에서는 GUI 환경에 대해서만 설명한다.

포스트를 작성하는 시점(2020.09)을 기준으로 emacs 에디터의 버전은 emacs 26.3 을 사용하였으며 폰트는 Hack 을 사용하였고 테마는 doom-dark+ 테마를 사용하였다. 포스트에서 설명하는 모든 내용들은 우분투 18.04 LTS 환경에서 테스트하였다. 설정 파일은 다음 링크에서 다운받을 수 있다. emacs(dev)

2 How to setup

sudo apt install emacs 를 통해 emacs를 설치하는 경우 일반적으로 26.3 버전이 아닌 다른 버전의 emacs가 설치될 것이다. 필자는 26.3 보다 낮은 버전에서 패키지를 설치할 때 TLS 에러 때문에 설치를 못했던 경험이 있어서 되도록이면 소스코드를 통해 빌드하는 방법을 추천한다. 물론 apt 를 통해 설치해도 최신 버전의 emacs가 설치되므로 큰 문제는 발생하지 않을 것이다.

필자의 설치 방법을 소개한다. 우선 Documents 디렉토리로 이동한다. 

$ cd ~/Documents

다음으로 아래 명령을 통해 설치 스크립트를 다운로드한다. 또는 해당 링크 에서 스크립트를 Documents 폴더로 복사한다. 

$ wget https://raw.githubusercontent.com/edward0im/lockdpwn/master/shell_scripts/install_emacs.sh

해당 스크립트를 열어서 readonly version 변수가 26.3 으로 되어 있는지 확인한다. 다음으로 해당 스크립트에 실행 권한을 준 후 실행하여 emacs 26.3을 설치한다. 

$ chmod 755 ./install_emacs.sh
$ ./install_emacs.sh

다음으로 emacs(dev) 환경설정 파일을 다운로드 받아서 홈폴더로 이동시킨다. 이 때, emacs 파일 이름 앞에 점(.)을 추가해야 정상적으로 emacs에서 인식한다. 

# Move .emacs to home folder.
$ mv  ~/Downloads/emacs_dev  ~/.emacs

그리고 다음 경로로 이동하여 emacs를 실행한다. emacs를 실행하면 .emacs 파일에서 패키지 목록을 읽어서 패키지들을 자동으로 설치한다. 이 때, TLS 관련 질문이 나오면 always 를 선택하여 모든 패키지들을 설치한다. 

$ cd ~/Documents/emacs-26.3/src/

# Run emacs 26.3
$ ./emacs

여기까지 완료되었으면 모든 emacs 기능들이 정상적으로 설치되었다.

3 My emacs configuration

해당 섹션에서는 필자가 사용하는 emacs의 다양한 기본 기능 및 플러그인 기능들에 대해 설명한다. 사용하는 .emacs 파일은 emacs(dev) 에서 다운로드 받을 수 있다. 해당 키바인딩은 필자가 임의로 설정한 키바인딩이므로 원한다면 얼마든지 바꿔서 사용할 수 있다.

3.1 General Features

emacs는 기본 키바인딩들이 대부분 Control로 시작하여 왼쪽 새끼손가락에 무리가 많이 가기 때문에 기본 키바인딩을 거의 대부분 Space 키부터 시작하도록 변경하였다. Custom keybindings 섹션의 경우 emacs 자체의 기본 키바인딩을 변경한 필자의 키바인딩에 대해 설명하였다. emacs에는 수많은 기능 및 키바인딩들이 존재하지만 이를 다 설명하지 않고 실제로 자주 사용하는 키바인딩 위주로 설명하였다.

Control 키바인딩은 <C-{...}> 과 같이 표현하고 Alt 키바인딩은 <A-{...}> 으로 표현하였다. 그리고 Space 키바인딩은 <SPC-{...}> 로 표현하였다.

3.1.1 Custom keybindings

에디터의 필수적인 키바인딩(저장, 파일열기, 파일닫기, 다른창 열기 등)은 evil 모드를 사용하여 대부분이 vim의 키바인딩과 동일하다.

3.2 Package Features

3.2.1 helm

emacs의 GUI를 코딩에 최적화되도록 변경해주는 패키지. 주로 버퍼창을 쉽게 네비게이션하는데 사용한다.

3.2.2 projectile

cmake를 통해 관리되는 프로젝트들을 자동으로 인식하여 구분해주는 패키지

3.2.3 magit

git 관련 명령을 수행할 수 있는 패키지

3.2.4 evil

vim 키바인딩을 사용할 수 있는 패키지

3.2.5 yasnippet

특정 키워드를 입력 후 Tab을 누르면 미리 저장된 Snippet들이 나오는 패키지

3.2.6 lsp-mode

코드 네비게이션(함수 이동, 에러 체크) 등을 수행할 수 있는 패키지

3.2.7 lsp-ui

lsp-mode에 다양한 GUI 효과들을 사용할 수 있는 패키지

3.2.8 eldoc

emacs 최하단 상태창에 커서 위의 있는 함수 또는 변수의 정보를 표시하는 패키지

3.2.9 doom-modeline

doom 테마의 modeline을 사용할 수 있는 패키지

3.2.10 flycheck

코드의 에러 구문을 체크하는 패키지

3.2.11 neotree

vim의 NerdTree와 같이 현재 파일의 디렉토리 구조를 볼 수 있는 패키지

3.2.12 eyebrowse

하나의 emacs 창에서 여러 workspace를 사용하여 여러 코드들을 편집할 수 있는 패키지

3.2.13 git-gutter

git 프로젝트의 경우 코드의 추가/변경/삭제 내역을 줄번호 옆에 표시해주는 패키지

3.2.14 centaur-tabs

emacs 상단의 Tab 기능을 사용할 수 있는 패키지

3.2.15 which-key

특정 키바인딩을 입력 중인 경우 사용 가능한 키바인딩을 팝업창으로 보여주는 패키지

3.2.16 which-func

현재 커서 위에 놓여진 함수의 이름을 modeline에 표시하는 패키지

3.2.17 multi-term

emacs 내에서 터미널(bash, zsh)를 사용할 수 있는 패키지

3.2.18 iedit

특정 단어를 코드 전체에 걸쳐서 일괄적으로 변경할 수 있는 패키지

3.2.19 wgrep

grep 명령어를 통해 동일한 단어를 검색한 경우 이를 일괄적으로 수정할 수 있는 패키지

3.2.20 avy

코드 간 빠른 네비게이션 기능을 수행할 수 있는 패키지

3.2.21 ecb

코딩에 필요한 현재 디렉토리, 현재 파일, 함수 또는 클래스 정보를 표시해주는 GUI 패키지

3.2.22 highlight-symbol

특정 코드를 하이라이팅하는 기능, 같은 단어를 빠르게 네비게이션하는 기능을 제공하는 패키지

3.2.23 highlight-indentation

코드 들여쓰기를 시각적으로 표현해주는 패키지

3.2.24 google-c-style

구글의 C 코딩 스타일 들여쓰기를 적용해주는 패키지

3.2.25 undo-tree

코드의 작업취소(undo), 복구(redo)를 수행해주는 패키지. 시각적으로 변경 정보를 한 눈에 볼 수 있는 visual undo-tree 기능도 제공한다.

3.2.26 mic-paren

괄호에 커서를 올리면 반대편 괄호를 하이라이팅해주는 패키지

3.2.27 ws-butler

White Space를 자동으로 제거해주는 패키지

3.2.28 hide-comnt

코멘트를 일시적으로 숨겨주는 패키지

3.2.29 smartparens

괄호를 입력하면 닫는괄호도 자동으로 입력해주는 패키지

3.2.30 jedi

Python 코드 네비게이션 패키지

3.2.31 elpy

3.2.32 cmake-ide

cmake 관련 명령을 수행하는 패키지

3.2.33 cmake-mode

CmakeLists.txt 파일을 cmake 구문에 맞게 하이라이팅해주는 패키지
