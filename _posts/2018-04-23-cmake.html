Introduction

cmake는 리눅스, 윈도우, 맥 등 운영체제에 관계없이 하나의 코드만으로 실행 파일을 생성해주는 크로스 컴파일러 프로그램이다. 처음에 익숙해지기까지 알아야 될 정보들이 많지만 한 번 익숙해진다면 운영체제에 독립적으로 개발할 수 있기 때문에 매우 편리하고 효율적으로 작업할 수 있다. 해당 포스트는 TUWLAB님의 블로그의 내용을 참고하였다.

Commands

cmake_minimum_required() 필요 CMake 최소 버전 명시

project() 프로젝트 이름 설정

message() 콘솔에 메세지 출력 

$ message ( [<Type>] <메시지> )
ex: $ message ( FATAL_ERROR "Fatal error occurred!" )
  • Type은 다음중 하나이며, 생략할 수 있다.
    • STATUS : 상태 메시지 출력 (메시지 앞에 '–'가 추가되서 출력됨)
    • WARNING : 경고 메시지를 출력하고, 계속 진행
    • AUTHOR_WARNING : 프로젝트 개발자용 경고 메시지를 출력하고, 계속 진행
    • SEND_ERROR : 오류 메시지를 출력하고 계속 진행하지만, Makefile 생성은 하지 않음
    • FATAL_ERROR : 오류 메시지를 출력하고, 작업을 즉시 중단

add_executable() 빌드 최종 결과물로 생성할 실행 파일을 추가한다. 이 명령을 반복하여 생성할 실행 파일을 계속 추가할 수 있다. 

$ add_executable ( app.out main.c foo.c bar.c )

add_library() 빌드 최종 결과물로 생성할 라이브러리를 추가한다. 이 명령을 반복하여 생성할 라이브러리를 계속 추가할 수 있다. 

$ add_library ( <라이브러리_이름> [STATIC|SHARED|MODULE] <소스_파일> <소스_파일> ... )
ex: $ add_library ( app STATIC foo.c bar.c )

<라이브러리_이름> : 생성할 라이브러리 이름 (lib~.a / lib~.so 에서 ~에 들어갈 값) [STATIC|SHARED|MODULE] : 라이브러리 종류 (생략시 STATIC) <소스_파일> : 라이브러리를 생성하는 데 필요한 소스 파일

add_custom_target() 사용자 정의 Target 추가 통상적인 빌드 절차로 생성할 수 없는 Target을 추가한다.. 라지만, 실상은 make 매크로를 정의할 때 더 많이 사용한다.(make flash 라던가.) Taget의 Recipe를 직접 지정해야 하는 경우 유용하게 활용할 수 있다. 

$ ADD_CUSTOM_TARGET (
                <이름> [ALL]
                [COMMENT <출력_메시지>]
                [DEPENDS <의존_대상_목록>]
                [WORKING_DIRECTORY <작업_디렉토리>]
                COMMAND <명령>
                [COMMAND <명령>]
                ...
)

ex:
$ add_custom_target ( flash
        COMMENT "Flashing binary"
        WORKING_DIRECTORY ${CMAKE_RUNTIME_OUTPUT_DIRECTORY}
        COMMAND python esptool.py write_flash app.bin
        DEPENDS app.out
        )

add_dependencies() Top-level Target간의 의존성을 지정한다. Top-level Target이란 add_executable, add_library, add_custom_target 명령으로 정의한 Target들을 의미한다. 

$ add_dependencies ( <Target_이름> <의존_대상> <의존_대상> ... )
$ add_dependencies ( flash app.out )

install() Makefile에서 관용적으로 설치용 Target으로 사용되는 install target의 동작 방식을 정의한다. 즉, make install 명령을 실행했을 때 무슨 동작을 수행할지를 지정한다. 

$ install ( TARGETS <Target_목록>
        RUNTIME DESTINATION <바이너리_설치_경로>
        LIBRARY DESTINATION <라이브러리_설치_경로>
        ARCHIVE DESTINATION <아카이브_설치_경로>
        )

ex: $ install ( TARGETS app.out app
        RUNTIME_DESTINATION /usr/local/bin
        ARCHIVE_DESTINATION /usr/local/lib
        )
예: 위 명령은 app.out은 /usr/local/bin에, libapp.a는 /usr/local/lib에 설치하는 install target을 추가한다.

Variables

변수를 설정하는 방법은 set($VARIABLE value) 로 설정할 수 있다. 변수를 확인하는 방법은 message(${VARIABLE}) 로 확인할 수 있다.

${CMAKE_PROJECT_NAME} 프로젝트 이름

${CMAKE_BUILD_TYPE} 빌드 형상(Configuration) Debug : 디버깅 목적의 빌드 Release : 배포 목적의 빌드 RelWithDebInfo : 배포 목적의 빌드지만, 디버깅 정보 포함 MinSizeRel : 최소 크기로 최적화한 배포 목적 빌드

${CMAKE_VERBOSE_MAKEFILE} Verbose Makefile 작성 여부 이 변수는 Switch변수이며, 다음과 같이 값을 true(또는 1)로 지정하면 빌드 상세 과정을 모두 출력하는 Makefile을 생성한다. 

$ set ( CMAKE_VERBOSE_MAKEFILE true )

${CMAKE_INSTALL_PREFIX} 설치 매크로(make install)에서 실행 바이너리와 라이브러리 등의 최종 생성물을 복사할 설치 디렉토리를 지정한다. install() 명령에서 상대 경로를 사용한 경우, 이 변수에 지정한 디렉토리가 Base 디렉토리가 됩니다. 

$ set ( CMAKE_INSTALL_PREFIX /usr/bin )
  • 이 변수를 별도로 지정하지 않으면 기본값은 /usr/local 이다.
